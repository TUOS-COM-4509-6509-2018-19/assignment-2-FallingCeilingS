Classification¶

  • In classification we take in a feature matrix and make predictions of class labels given the features.
  • Our features are xixi for the iith data point
  • Our labels are yiyi which is either -1 (negative) or +1 (positive).

Classification¶

  • predict the class label, yiyi, given the features associated with that data point, xixi, using the prediction function:

    f(xi)=sign(w⊤xi+b)f(xi)=sign(w⊤xi+b)

  • Decision boundary for the classification is given by a hyperplane.

  • Vector ww is the normal vector) to the hyperplane.
  • Hyperplane is described by the formula w⊤x=−bw⊤x=−b

Drawing Decision Boundary¶

The decision boundary is where the output of the function changes from -1 to +1 (or vice versa) so it's the point at which the argument of the signsign function is zero. So in other words, the decision boundary is given by the line defined by x1w1+x2w2=−bx1w1+x2w2=−b (where we have dropped the index ii for convenience). In this two dimensional space the decision boundary is defined by a line. In a three dimensional space it would be defined by a plane and in higher dimensional spaces it is defined by something called a hyperplane. This equation is therefore often known as the separating hyperplane because it defines the hyperplane that separates the data. To draw it in 2-D we can choose some values to plot from x1x1 and then find the corresponding values for x2x2 to plot using the rearrangement of the hyperplane formula as follows

x2=−(b+x1w1)w2x2=−(b+x1w1)w2

Of course, we can also choose to specify the values for x2x2 and compute the values for x1x1 given the values for x2x2,

x1=−b+x2w2w1x1=−b+x2w2w1

Switching Formulae¶

It turns out that sometimes we need to use the first formula, and sometimes we need to use the second. Which formula we use depends on how the separating hyperplane leaves the plot.

We want to draw the separating hyperplane in the bounds of the plot which is showing our data. To think about which equation to use, let's consider two separate situations (actually there are a few more).

  1. If the separating hyperplane leaves the top and bottom of the plot then we want to plot a line with values in the yy direction (given by x2x2) given by the upper and lower limits of our plot. The values in the xx direction can then be computed from the formula for the plane.

  2. Conversely if the line leaves the sides of the plot then we want to plot a line with values in the xx direction given by the limits of the plot. Then the values in the yy direction can be computed from the formula. Whether the line leaves the top/bottom or the sides of the plot is dependent on the relative values of w1w1 and w2w2.

This motivates a simple if statement to check which situation we're in.

# %load -s update_perceptron mlai.py
def update_perceptron(w, b, x_plus, x_minus, learn_rate):
    "Update the perceptron."
    # select a point at random from the data
    choose_plus = np.random.uniform(size=1)>0.5
    updated=False
    if choose_plus:
        # choose a point from the positive data
        index = np.random.randint(x_plus.shape[0])
        x_select = x_plus[index, :]
        if np.dot(w, x_select)+b <= 0.:
            # point is currently incorrectly classified
            w += learn_rate*x_select
            b += learn_rate
            updated=True
    else:
        # choose a point from the negative data
        index = np.random.randint(x_minus.shape[0])
        x_select = x_minus[index, :]
        if np.dot(w, x_select)+b > 0.:
            # point is currently incorrectly classified
            w -= learn_rate*x_select
            b -= learn_rate
            updated=True
    return w, b, x_select, updated

How the algorithm works?¶

  • We want to find parameters ww and bb that allow a correct prediction for a datapoint xixi.

  • We can then run an algorithm with the following rules:

    • If the vector xixi is correctly classified using ww and bb, we don't change those parameters.

    • If the vector xixi is incorrectly classified using ww and bb, we change those parameters by adding a correction terms.

What is the correction term that we add?¶

  • If the vector xixi is incorrectly classified as negative being positive, we add ηxiηxi to ww and ηη to bb.
  • If the vector xixi is incorrectly classified as positive being negative, we substract ηxiηxi to ww and ηη to bb.
  • ηη is known as the learning rate.

Why this works?¶

  • As an example, say xixi has label yi=+1yi=+1, but ww and bb are such that the prediction is f(xi)=−1f(xi)=−1

    f(xi)=sign(w⊤xi+b)=−1f(xi)=sign(w⊤xi+b)=−1

  • The above implies that w⊤xi+b<0w⊤xi+b<0

  • Now we apply the correction terms to ww and bb. Let's call the new values for ww and bb, wnewwnew and bnewbnew,

    \begin{align*}

    \mathbf{w}_{\text{new}} & = \mathbf{w} + \eta \mathbf{x}_i\\
                      b_{\text{new}} & = b + \eta

    \end{align*}

  • The new predicted value will be

    \begin{align} f(\mathbf{x}i) & = \text{sign}(\mathbf{w}{\text{new}}^{\top}\mathbf{x}i + b{\text{new}}) \

                 &  = \text{sign}((\mathbf{w}+\eta \mathbf{x}_i)^{\top}\mathbf{x}_i +  b + \eta)\\
             &  = \text{sign}(\mathbf{w}^{\top}\mathbf{x}_i + b +\eta \mathbf{x}_i^{\top}\mathbf{x}_i + \eta)

    \end{align}

  • Notice how the term ηx⊤ixi+ηηxi⊤xi+η (strictly positive) pushes the argument of f(⋅)f(⋅) to become positive and making the right prediction.

Steepest Descent¶

  • Minimize the sum of squares error function.
  • One way of doing that is gradient descent.
  • Initialize with a guess for mm and cc
  • update that guess by subtracting a portion of the gradient from the guess.
  • Like walking down a hill in the steepest direction of the hill to get to the bottom.

Algorithm¶

  • We start with a guess for mm and cc.

Deriving the Gradient¶

To see how the gradient was derived, first note that the cc appears in every term in the sum. So we are just differentiating (yi−mxi−c)2(yi−mxi−c)2 for each term in the sum. The gradient of this term with respect to cc is simply the gradient of the outer quadratic, multiplied by the gradient with respect to cc of the part inside the quadratic. The gradient of a quadratic is two times the argument of the quadratic, and the gradient of the inside linear term is just minus one. This is true for all terms in the sum, so we are left with the sum in the gradient.

Slope Gradient¶

The gradient with respect tom mm is similar, but now the gradient of the quadratic's argument is −xi−xi so the gradient with respect to mm is

dE(m,c)dm=−2∑i=1nxi(yi−mxi−c)dE(m,c)dm=−2∑i=1nxi(yi−mxi−c)

which can be implemented in python (numpy) as

In [20]:

Update Equations¶

  • Now we have gradients with respect to mm and cc.
  • Can update our inital guesses for mm and cc using the gradient.
  • We don't want to just subtract the gradient from mm and cc,
  • We need to take a small step in the gradient direction.
  • Otherwise we might overshoot the minimum.
  • We want to follow the gradient to get to the minimum, the gradient changes all the time.

Stochastic Gradient Descent¶

  • If nn is small, gradient descent is fine.
  • But sometimes (e.g. on the internet nn could be a billion).
  • Stochastic gradient descent is more similar to perceptron.
  • Look at gradient of one data point at a time rather than summing across all data points)
  • This gives a stochastic estimate of gradient.

Stochastic Gradient Descent¶

The real gradient with respect to mm is given by

dE(m,c)dm=−2∑i=1nxi(yi−mxi−c)dE(m,c)dm=−2∑i=1nxi(yi−mxi−c)

but it has nn terms in the sum. Substituting in the gradient we can see that the full update is of the form

$$m\text{new} \leftarrow m\text{old} + 2\eta \left[x_1 (y1 - m\text{old}x1 - c\text{old})

 + (x_2 (y_2 -   m_\text{old}x_2 - c_\text{old}) + \dots + (x_n (y_n - m_\text{old}x_n - c_\text{old})\right]$$

This could be split up into lots of individual updates

m1←mold+2η[x1(y1−moldx1−cold)]m1←mold+2η[x1(y1−moldx1−cold)]
m2←m1+2η[x2(y2−moldx2−cold)]m2←m1+2η[x2(y2−moldx2−cold)]
m3←m2+2η[…]m3←m2+2η[…]
mn←mn−1+2η[xn(yn−moldxn−cold)]mn←mn−1+2η[xn(yn−moldxn−cold)]
